# 用户线程

运行在前台，执行具体的任务。

程序的主线程、连接网络的子线程等都是用户线程

# 守护线程

运行在后头，为其他前台线程服务

## 特点

一旦所有用户线程都结束运行，守护线程会随JVM一起结束工作

## 应用

- 垃圾回收线程
- 数据库连接池中的监测线程

**通过setDaemon(true)来表示一个线程属于守护线程**

## 注意

- setDaemon(true)必须在start方法之前调用，否则会抛出IllegalThreadStateException
- 在守护线程中产生的新线程也是守护线程
- 不是所有的任务都合适用守护线程来执行，比如读写操作或计算逻辑。
